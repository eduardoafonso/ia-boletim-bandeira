Solução Corporativa de IA
para Análise de Boletins de Bandeira

Avaliação de plataformas Microsoft para arquitetura de agente enterprise para processamento inteligente de documentos regulatórios das bandeiras Mastercard, Visa, Elo e Amex.

RAG / Document Intelligence Microsoft Copilot Studio Microsoft Foundry SharePoint + Power Automate Azure OpenAI Sistemas Legados PL/I
01 · Objetivo

O que a solução precisa resolver

Implementar um agente corporativo de IA especializado em análise documental para processamento de boletins de bandeira, operando sobre um repositório SharePoint com documentos PDF e DOC versionados por bandeira, boletim e vigência.

  • Identificação de mudanças mandatórias
  • Detecção de alterações de layout
  • Análise comparativa entre versões
  • Geração automatizada de checklists de teste
  • Apoio à análise de impacto funcional
  • Rastreabilidade documental end-to-end
  • Suporte a sistemas legados em PL/I
  • Classificação de criticidade regulatória

Contexto Técnico

Cenário de Enterprise Document Intelligence com RAG, automação regulatória e governança documental corporativa integrada a sistemas críticos.

Repositório Principal

Microsoft SharePoint como fonte de verdade para PDFs e DOCs versionados. Gatilhos automáticos via Power Automate a cada novo upload.

Escopo de Usuários

Operação para ~16 usuários com perfis distintos (analistas, desenvolvedores PL/I, gestores). RBAC e segregação por papel são requisitos não-negociáveis.

03 · Fluxo Operacional

Pipeline de processamento proposto

Arquitetura de operação do agente do upload do boletim até a entrega dos artefatos para os times de desenvolvimento e sustentação.

1

Upload do boletim no SharePoint

Publicação do documento PDF ou DOC no repositório SharePoint, organizado por bandeira, número do boletim e vigência. Versionamento automático habilitado.

SharePoint Metadados Versionamento
2

Trigger automático via Power Automate

Detecção de novos uploads ou atualizações. Extração de metadados (bandeira, versão, data de vigência) e disparo do pipeline de processamento.

Power Automate Event-driven Metadados
3

Extração e pré-processamento documental

Conversão do documento para texto estruturado via Azure AI Document Intelligence (OCR, extração de tabelas, campos). Chunking semântico para indexação no vetor de embeddings.

Azure AI Document Intelligence OCR + Tabelas Chunking semântico
4

Análise pelo agente (Copilot Studio e/ou Microsoft Foundry) + Azure OpenAI

O agente executa a análise inteligente do boletim com base no contexto extraído e nas versões anteriores indexadas.

Copilot Studio Microsoft Foundry Azure OpenAI RAG

Atividades executadas pelo agente

  • Identificação de nova versão e comparação com versão anterior
  • Detecção de mudanças mandatórias e alterações de layout
  • Extração de regras, campos e fluxos impactados
  • Classificação de impacto: Crítico / Alto / Médio / Baixo
  • Mapeamento de sistemas e programas PL/I potencialmente impactados
5

Geração de artefatos estruturados

O agente produz automaticamente os documentos de trabalho para os times de desenvolvimento e QA.

Checklist de testes Análise funcional Resumo executivo Rastreabilidade
6

Publicação e distribuição organizacional

Resultados publicados de forma estruturada no SharePoint e notificação distribuída via Teams para os grupos responsáveis, com links de acesso direto e urgência classificada.

Teams SharePoint Azure DevOps (backlog)

SharePoint

  • Atua como repositório corporativo central dos boletins, versões e documentos relacionados
  • Permite versionamento nativo, controle documental e governança de acesso integrada ao ecossistema Microsoft 365
  • Serve como fonte primária para ingestão, indexação e rastreabilidade dos conteúdos processados pelo pipeline de IA
  • Sem ele, não existe uma camada estruturada de armazenamento corporativo com histórico auditável e integração nativa com os agentes

Copilot Studio

  • É a plataforma low-code responsável pela construção, orquestração e publicação do agente conversacional corporativo
  • Permite integrar Azure AI Search, fluxos do Power Automate e serviços cognitivos em uma experiência unificada
  • Centraliza intents, prompts, memória contextual e regras de interação do agente especializado em compliance e boletins
  • Sem ele, não existe uma camada de experiência conversacional corporativa integrada aos serviços de IA da Microsoft sem desenvolvimento customizado

Microsoft Foundry

  • É a plataforma pro-code para construir, hospedar e orquestrar agentes de IA com controle total sobre modelo, lógica e infraestrutura
  • Permite acessar um catálogo com milhares modelos (OpenAI, Llama, DeepSeek, Grok, Mistral, entre outros), além de orquestração multiagente nativa
  • Não tem cobrança de plataforma — o custo de tokens de modelo e ferramentas/conectores usados pelos agentes
  • Sem ele, a solução fica limitada aos modelos e à lógica de orquestração nativos do Copilot Studio, sem o controle pro-code necessário para arquiteturas multiagente mais complexas

Power Automate

  • Atua como camada de automação e orquestração entre SharePoint, serviços de IA e processos corporativos
  • Permite disparar fluxos automáticos para ingestão, classificação, indexação e atualização de documentos
  • Garante integração event-driven entre os componentes do ecossistema sem necessidade de desenvolvimento excessivo
  • Sem ele, os processos dependeriam de execução manual ou integrações customizadas de maior complexidade operacional

Azure OpenAI (via Foundry Models)

  • É o modelo de linguagem responsável pela interpretação contextual, reasoning e geração das respostas do agente
  • Permite sumarização, comparação semântica, extração de insights e análise avançada dos boletins regulatórios
  • Trabalha em conjunto com o Azure AI Search no padrão RAG para reduzir alucinação e aumentar precisão contextual
  • Sem ele, o sistema perde capacidade analítica avançada e interação em linguagem natural orientada ao domínio

Azure AI Document Intelligence

  • É o serviço responsável pela extração estruturada de informações a partir de PDFs, imagens e documentos não estruturados
  • Permite identificar tabelas, campos, layouts e conteúdos relevantes antes do processo de indexação vetorial
  • Reduz esforço manual de tratamento documental e aumenta qualidade dos dados enviados ao mecanismo RAG
  • Sem ele, documentos complexos exigiriam parsing customizado e maior intervenção humana no pré-processamento

Azure AI Search (Foundry IQ)

  • É o mecanismo de índice vetorial que viabiliza o RAG sobre os boletins indexados
  • Permite busca semântica e comparação entre versões com contexto enriquecido
  • Se integra nativamente ao Copilot Studio e ao Microsoft Foundry como fonte de conhecimento customizada
  • Sem ele, o agente não consegue comparar versões de forma estruturada e rastreável

Microsoft Graph API

  • Atua como camada unificada de integração com serviços do Microsoft 365 e diretórios corporativos
  • Permite acessar usuários, grupos, permissões, arquivos, calendários e informações organizacionais de forma centralizada
  • Viabiliza personalização contextual do agente com base em identidade, perfil e permissões do usuário
  • Sem ela, integrações entre o agente e o ecossistema Microsoft dependeriam de múltiplas APIs isoladas e maior complexidade técnica

Azure DevOps

  • Atua como plataforma de gestão do ciclo de vida da aplicação, pipelines CI/CD e governança de entregas
  • Permite automação de build, deploy, testes e gerenciamento ágil das demandas do projeto
  • Integra desenvolvimento, operações e versionamento em um fluxo controlado e auditável
  • Sem ele, a esteira de entrega perde automação, rastreabilidade operacional e padronização de deployment corporativo

GitHub

  • É a plataforma de versionamento e colaboração utilizada para gestão do código-fonte, prompts e artefatos da solução
  • Permite controle de branches, revisão técnica, rastreabilidade de mudanças e automação de pipelines DevOps
  • Centraliza documentação técnica, infraestrutura como código e versionamento dos componentes de IA
  • Sem ele, o ciclo de desenvolvimento perde governança, colaboração estruturada e controle eficiente de evolução da solução

Microsoft 365 Copilot

  • Atua como camada de produtividade inteligente integrada ao ecossistema Microsoft 365
  • Permite consumir informações corporativas, documentos e insights diretamente em aplicações como Teams, Outlook, Word e Excel
  • Pode utilizar os agentes especializados desenvolvidos no Copilot Studio ou no Microsoft Foundry como extensões contextuais de negócio
  • Sem ele, os usuários perdem a experiência de IA integrada ao fluxo operacional diário dentro das ferramentas corporativas

Azure Active Directory (Entra ID)

  • É o serviço de identidade e autenticação responsável pelo controle de acesso corporativo da solução
  • Permite autenticação centralizada, Single Sign-On (SSO) e aplicação de políticas de segurança baseadas em identidade
  • Garante segregação de acesso aos boletins, agentes e recursos cognitivos conforme perfil organizacional
  • Sem ele, a solução perde governança de identidade corporativa e controle seguro de autenticação e autorização

API Gateway / Azure Logic Apps

  • Atua como camada de integração, orquestração e exposição controlada dos serviços e APIs da solução
  • Permite conectar sistemas internos e externos com baixa complexidade de desenvolvimento e suporte a workflows enterprise
  • Facilita controle de autenticação, transformação de payloads, governança de integrações e monitoramento operacional
  • Sem ele, integrações entre sistemas dependeriam de implementações customizadas com maior acoplamento e menor governança arquitetural

Protocolo A2A (Agent-to-Agent)

  • Protocolo (em preview) que permite comunicação direta entre agentes independentes, possibilitando arquiteturas multiagente onde cada agente tem uma responsabilidade especializada
  • Viabiliza que agentes especializados do pipeline se comuniquem entre si dentro do Microsoft Foundry, em vez de uma lógica monolítica única
  • Permite que agentes construídos em frameworks diferentes (Agent Framework, LangGraph, SDKs de terceiros) troquem tarefas e resultados de forma padronizada
  • Sem ele, a orquestração entre agentes especializados dependeria de integrações customizadas e ponto-a-ponto, aumentando o acoplamento e a complexidade de manutenção

Variáveis que impactam o custo

  • Volume de documentos processados por mês (número de boletins e páginas)
  • Quantidade de Copilot Credits consumidos pelo agente (Copilot Studio) ou volume de tokens (Foundry)
  • Uso de OCR para documentos digitalizados (Azure AI Document Intelligence)
  • Tamanho do índice vetorial no Azure AI Search / Foundry IQ
  • Licenças M365 já existentes na organização (redução de custo potencial)
  • Escolha do modelo de IA — modelos menores (mini/nano) custam significativamente menos por token do que os modelos de maior capacidade

07 · Riscos e Premissas

O que precisa estar endereçado

Riscos identificados

  • Qualidade do OCR para boletins digitalizados ou de baixa resolução
  • Variação de layout entre bandeiras pode exigir pipelines diferenciados
  • Alucinação do LLM em regras regulatórias críticas — exige validação humana
  • Dependência de conectividade e disponibilidade dos serviços Azure
  • Curva de adoção pelos times de desenvolvimento PL/I
  • Gestão do modelo de custos variáveis conforme crescimento do volume
  • Governança de dados sensíveis dentro do Azure OpenAI (LGPD)
  • Se optar por Microsoft Foundry, necessidade de time com competência pro-code para manter a solução

Premissas e pré-requisitos

  • Licenças Azure e M365 provisionadas ou elegíveis para aquisição
  • SharePoint configurado com estrutura de pastas e metadados definidos
  • Azure OpenAI aprovado na subscrição (requer aprovação da Microsoft)
  • Equipe com capacidade de configurar Power Automate + Copilot Studio, e/ou desenvolvedores para Microsoft Foundry
  • Definição de RBAC e grupos de usuários no Entra ID
  • Inventário mínimo dos sistemas PL/I e domínios cobertos por cada bandeira
  • Processo de validação humana definido antes da publicação dos artefatos
08 · Conclusão Executiva

Recomendação final

Para um agente corporativo de análise de boletins de bandeira com os requisitos mapeados, a arquitetura recomendada combina:

❌

Copilot Chat

Não recomendado como plataforma principal. Útil apenas para POCs rápidas e validação de prompts em fase exploratória.

🟡

Microsoft 365 Copilot

Recomendado como camada complementar de produtividade, busca contextual e apoio operacional diário para os usuários.

✅

Copilot Studio

Boa aderência técnica e corporativa em modelo low-code. Combinado com Azure OpenAI, AI Search e Power Automate forma um pipeline completo com menor esforço de engenharia.

🔷

Microsoft Foundry

Recomendado como núcleo de IA/RAG quando há time de desenvolvimento, ou como evolução futura para arquiteturas multiagente e maior portabilidade de modelo.
